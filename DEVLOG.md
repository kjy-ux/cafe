# ☕ 카페 앱 개발 과정 요약

이 문서는 `BLUEPRINT.md`(청사진)를 기반으로 카페 앱을 구현하며 진행된 개발 과정, 논의된 결정 사항, 발견하여 수정한 버그들을 정리한 기록입니다.

---

## 0. 시작 — 청사진 기준 확립

- 강사님이 만든 `BLUEPRINT.md`가 유일한 기준 문서이며, 별도 채점 루브릭 없음
- **변경 불가**: 폴더/파일 구조, 코로케이션 원칙(HTML-CSS-JS 동일 위치·이름 매칭), 고객/관리자 역할 분리, 기존 함수·변수·CSS 변수의 이름과 의미
- **자유롭게 가능**: 마크업 내용, 스타일 디테일, 로직 구현, 공유 자원에 **새 항목 추가**
- 과제 목적은 "기본기(HTML/CSS/JS, localStorage) 숙달 확인"이지만, 발표가 예정되어 있어 완성도(폰트 로딩, 인터랙션, 토스트 등)도 함께 챙기기로 함

---

## 1. 1단계: 공유 자원 검토 및 보강

기존 코드 재검토 중 발견한 문제:
- `utils.js` 주석 인코딩 깨짐 (`포�팅` → `포맷팅`)
- `data.js`의 말차 라떼 설명에 불필요한 영문/공백 혼입
- id 16 메뉴명 오타 `'체리즈'` → 설명(겉바속촉)과 맞지 않아 `'츄러스'`로 수정

발표 완성도를 위해 추가한 것:
- Google Fonts `Noto Sans KR` 실제 로딩 (표준 `<head>` 스니펫으로 문서화)
- `--transition-fast`/`--transition-base` 토큰, 토스트 알림 컴포넌트(`showToast`)
- `☕` 이모지 파비콘 (data URI, 이미지 파일 불필요)
- `--color-success`, `:focus-visible`, `::selection`, `.page-container` 공유 클래스
- 주문 상태 배지 시스템(`.badge--*`, `getStatusBadgeClass()`) — 6개 상태별 색상
- 메뉴 CRUD용 localStorage 함수(`getMenus`/`addMenu`/`updateMenu`/`deleteMenu`), `addToCart`가 `MENU_ITEMS` 대신 `getMenuById`를 참조하도록 수정 (관리자 메뉴 변경사항과 정합성 유지)
- `escapeHtml()`, `hasBatchim()`/`josa()`(한글 조사 을/를 자동 처리) 유틸

## 2. 2단계: 관리자 — 메뉴 관리 (list/detail/create/edit)

- 카테고리 필터, 상세 보기, 생성/수정 폼(이미지 URL 미리보기 포함) 구현
- **발견 버그 1**: `list.js`에서 사용자 입력(메뉴 이름 등)이 이스케이프 없이 `innerHTML`에 삽입 → XSS 위험. `escapeHtml()` 적용 후 실제 `<script>` 삽입 시도로 재현·검증
- **발견 버그 2**: `.image-preview { display: flex }` 같은 컴포넌트 클래스가 브라우저 기본 `[hidden]{display:none}`보다 우선 적용되어, `hidden` 속성을 줘도 안 숨겨지는 문제. 개별 예외 대신 `variables.css`에 `[hidden]{display:none !important}` 전역 규칙으로 근본 해결

## 3~8단계: 나머지 고객·관리자 페이지 순차 구현

| 단계 | 내용 |
|---|---|
| 3단계 | 고객 메뉴 조회 (목록/상세, 장바구니 담기) |
| 4단계 | 장바구니 (수량 조절, 삭제, 주문하기) |
| 5단계 | 고객 주문내역 (목록/상세) |
| 6단계 | 메인 페이지 (히어로 + 인기 메뉴 미리보기) |
| 7단계 | 마이페이지 (주문 통계 + 최근 주문, 로그인 기능 없어 주문 데이터 기반으로 구성) |
| 8단계 | 관리자 대시보드 & 주문 관리 (상태 필터, 인라인 상태 변경) |

- **발견 버그**: 장바구니 배지를 헤더에 추가하면서 모바일에서 내비게이션 텍스트가 글자 중간에서 줄바꿈되는 문제 → `white-space: nowrap` + 헤더를 세로로 쌓는 방식으로 3개 페이지 동시 수정
- 관리자에서 주문 상태를 바꾸면 고객 측 주문내역/마이페이지에 실시간 반영되는 것까지 확인

8단계 완료 시점에 청사진 체크리스트 50/50 전부 완료 → 전체 14개 페이지 콘솔 에러·내부 링크·하드코딩 색상·XSS 이스케이프 감사 진행 (문제 없음 확인)

## 4. 실제 카페 앱 벤치마킹 — 확장 기능 4종

스타벅스·투썸플레이스 등 리서치 기반으로 4가지 기능 추가:

1. **메뉴 커스터마이징** — 온도(HOT/ICE)/사이즈(+500원)/샷 추가(+500~1000원), 카테고리별 적용 옵션 다름. 장바구니를 `lineId` 기반으로 재설계(옵션이 다르면 같은 메뉴도 별도 줄)
2. **재주문** — 주문내역/마이페이지에서 "다시 담기" → 옵션까지 그대로 재장바구니. 메뉴 삭제 시 부분/실패 처리
3. **메뉴 검색** — 카테고리 필터와 함께 동작하는 이름 검색
4. **즐겨찾기** — 하트 토글 + 마이페이지 "찜한 메뉴" 섹션

- **발견 버그**: `orders/detail.css`에 버튼 스타일이 아예 빠져 있어 "다시 담기" 버튼이 테두리만 있는 밋밋한 모습으로 렌더링 → 다른 페이지와 동일한 `.btn`/`.btn-primary` 추가

## 6. 레이아웃 구조 개편

폴더/코로케이션 구조는 유지하고 화면 구성만 변경:

- **관리자**: 상단 가로 헤더 → 왼쪽 고정 사이드바 (7개 페이지)
- **고객 메뉴 목록**: 상단 필터탭 → 왼쪽 sticky 사이드바(검색+카테고리) + 오른쪽 그리드
- **장바구니**: 세로 스택 → 왼쪽 아이템 목록 + 오른쪽 sticky 주문 요약

- **발견 버그**: 관리자 사이드바를 모바일에서 가로로 배치하며 "카페 관리자" 브랜드명이 글자 중간에서 줄바꿈 → 브랜드를 윗줄에 단독 배치하는 구조로 수정
