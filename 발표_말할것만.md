[PPT 1번 슬라이드 — 기획 의도]
안녕하세요, 카페 주문 웹앱, "카페" 프로젝트입니다.

첫 프로젝트라, 프론트엔드 기초를 다지려고
바닐라 HTML, CSS, JS로 직접 만들었습니다.

다들 같은 카페 앱에서 시작했지만,
AI랑 같이 디자인 방향을 계속 다듬어가면서
유럽풍 부티크 컨셉으로 전체 디자인을 다르게 가져갔습니다.

차별화 기능으로는 "나만의 레시피 저장, 원클릭 재주문"을 넣었습니다.
자주 마시는 옵션 조합에 이름을 붙여 저장하고,
한 번에 다시 담을 수 있게 만들었습니다.

[➡️ 다음 슬라이드 — PPT 2번, 기술스택]
기술스택은 HTML, CSS, 바닐라 자바스크립트, localStorage,
회원가입/로그인 기능이고,
Git으로 버전관리하고, GitHub Pages로 실제 배포까지 했습니다.

[➡️ 다음 슬라이드 — PPT 3번, 시연]
(여기서 PPT 끄고 브라우저로 넘어가서, 클릭하면서 한 단계씩 말하기)

이 조합을 나만의 레시피로 저장합니다.
장바구니에 담고, 주문 완료합니다.
마이페이지에서 즐겨찾기랑 저장한 레시피를 확인합니다.
회원가입, 로그인도 됩니다.
관리자 페이지는 로그인 가드로 막혀 있습니다.

[➡️ 브라우저에서 다시 PPT로 돌아와서 4번 슬라이드, 문제해결]
트러블슈팅은 네 가지입니다.

레시피 패널을 넣으니 카드가 밀려서, 분리해서 해결했습니다.
GitHub Pages에 배포하니 이미지가 다 깨져서,
경로를 동적으로 계산하도록 고쳤습니다.
로그인 도입 후 데이터가 공용이라, 계정별로 분리했습니다.
관리자 페이지가 뚫려있어서, 로그인 가드를 추가했습니다.
